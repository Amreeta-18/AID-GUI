GenderMag Automated Inclusivity Detector

What is The GenderMag Automated Inclusivity Detector?





This tool attempts to emulate a “real” GenderMag session and takes as input the data required when performing a GenderMag session, that is, the use-case, subgoal, and action, along with the HTML of the webpage for that step in the Cognitive Walkthrough.

From this input, AID extracts the sentence structure from the text in the subgoal and action using natural language process and checks for three decision rules, as follows:

Rule 1 (Cues to needed information): Keywords from subgoals and associated actions should be present on the webpage

The first rule in our set checks for situations where users like Abi (To learn more about the Abi persona, click here.) would not find all the information they need to complete their task. For example, the wording of the subgoal serves as the information that Abi seeks, and the words from actions serve as cues to direct Abi to a UI action. Without such cues, Abi would face difficulty finding all the information they need.

Rule 2 (Situating Abi in the context of the action performed): Linked pages should contain keywords from link labels

On clicking a link, the destination page should offer cues to help Abi (To learn more about the Abi persona, click here.) understand that they have reached the right place. If a project page fails to use words similar to what a link label hinted at, users like Abi could get confused.

Rule 3 (Information link navigation): Links should be labeled with a keyword or phrase

Users like Abi (To learn more about the Abi persona, click here.) click on a link only after gathering enough information and planning their next step. Labeled links provide Abi with information about the webpage they are supposed to visit. Other work agrees, recommending that links should have descriptive yet unique link labels and begin with keywords. Rule 3 captures this tenet.
